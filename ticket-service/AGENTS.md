# Ticket Service — Agent Instructions

Minimal NestJS 11 CRUD service for epics and tickets. Owns the PostgreSQL tables `epics` and `tickets`. Called by the MCP server's `create_epic` and `create_ticket` tools after the user accepts a plan.

No RabbitMQ, no Redis, no WebSocket.

## Stack

- **NestJS 11** — modules, controllers, services
- **TypeORM 0.3** — PostgreSQL, `synchronize: true` in dev
- **class-validator / class-transformer** — DTO validation via global `ValidationPipe`
- **uuid** — generated by the AI agent before persisting; accepted as-is

## Module structure

```
src/
  main.ts                               Bootstrap, ValidationPipe, port 8000
  app.module.ts                         Root module
  database/
    database.module.ts                  TypeORM root config (url from DATABASE_URL)
    entities/
      epic.entity.ts                    id (uuid, PK), name, requirements (jsonb), solution (jsonb), createdAt
      ticket.entity.ts                  id (uuid, PK), epicId (uuid), name, requirements (jsonb), acceptance_criteria (jsonb), createdAt
  epic/
    epic.module.ts
    controllers/
      epic.controller.ts                POST /api/epic · GET /api/epic/:id
    services/
      epic.service.ts                   create(), findOne()
    dto/
      create-epic.dto.ts                Validates CreateEpicDto (id, name, requirements, solution)
  ticket/
    ticket.module.ts
    controllers/
      ticket.controller.ts              POST /api/ticket · GET /api/epic/:epicId/tickets · GET /api/ticket/:id
    services/
      ticket.service.ts                 create(), findByEpic(), findOne()
    dto/
      create-ticket.dto.ts              Validates CreateTicketDto (id, epicId, name, requirements, acceptance_criteria)
  health/
    health.module.ts
    controllers/
      health.controller.ts              GET /api/health → { status: 'ok' }
```

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/epic/` | Create epic; `id` is provided by caller (agent-generated UUID) |
| `GET` | `/api/epic/:id` | Get epic by id |
| `POST` | `/api/ticket/` | Create ticket; `id` and `epicId` are provided by caller |
| `GET` | `/api/epic/:epicId/tickets` | Get all tickets for an epic, ordered by `createdAt ASC` |
| `GET` | `/api/ticket/:id` | Get ticket by id |
| `GET` | `/api/health` | `{ status: 'ok' }` |

## Entity shapes

```typescript
// epics table
Epic {
  id:           uuid (PK, provided by caller)
  name:         varchar(500)
  requirements: jsonb   // RequirementInterface[]
  solution:     jsonb   // SolutionInterface
  createdAt:    timestamptz (auto)
}

// tickets table
Ticket {
  id:                  uuid (PK, provided by caller)
  epicId:              uuid
  name:                varchar(500)
  requirements:        jsonb   // RequirementInterface[]
  acceptance_criteria: jsonb   // AcceptanceCriterionInterface[]
  createdAt:           timestamptz (auto)
}
```

IDs are generated by the AI agent's `plan_node` (using `uuid.uuid4()`) before calling MCP tools. The ticket-service stores them as provided.

## Environment

```
PORT=8000
DATABASE_URL=postgresql://tickets:tickets@localhost:5433/tickets
```

The ticket-service has its own PostgreSQL instance (`postgres-tickets`, host port 5433), separate from the backend's `postgres` instance. TypeORM creates both tables on startup via `synchronize: true`.
